# Design System

Use this file when the project has UI composition rules that are not fully explained by `.general_sdd/project/tech-stack.md`.

If the project adopts an external design system such as `shadcn/ui`, document that choice in `tech-stack.md` first. This file can then stay minimal or explicitly say that no additional local design system rules are needed.

## When To Fill This File

- Project-specific component hierarchy such as Atomic Design layers
- Storybook coverage, review, or state-modeling rules
- Naming, variant, spacing, and composition rules that extend the adopted UI library

## Project-Specific Rules

- Describe any local component taxonomy
- Describe Storybook authoring and review expectations
- Describe tokens, layout primitives, or accessibility rules that go beyond the base library
