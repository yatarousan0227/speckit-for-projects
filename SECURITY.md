# Security Policy

## Supported Versions

Security fixes are expected to land on the latest maintained code line.

At minimum, the following are intended to receive fixes:

- the latest release, if releases are being published
- otherwise, the current default branch

Older versions may not receive backports.

## Reporting A Vulnerability

Please do not open a public issue for a suspected vulnerability before maintainers have a chance to assess it.

Preferred reporting path:

1. Use the repository hosting platform's private security reporting feature if it is available.
2. If private security reporting is not available, contact the maintainers through a private channel.
3. If neither option is available, open a minimal public issue requesting a secure contact path and avoid posting exploit details.

Include the following when possible:

- affected version or commit
- impact summary
- reproduction steps
- proof-of-concept or logs, if safe to share
- any suggested mitigation

Maintainers will acknowledge receipt, assess severity, and coordinate disclosure after a fix or mitigation path is available.
